## Почему RSS Reader?

В мире сервисов по подписке и облачной синхронизации **RSS Reader** применяет другой подход. Это современное настольное приложение, которое хранит все ваши данные исключительно на вашем локальном компьютере. Работающее на базе Tauri и Rust, оно невероятно быстрое, экономно расходует память и тесно интегрируется с ИИ для улучшения вашего опыта чтения.

## Функции

### Чтение и управление лентами
- **Универсальная поддержка:** Удобная подписка на ленты RSS, Atom и JSON.
- **Умная синхронизация:** Инкрементальная синхронизация с использованием заголовков `ETag` и `Last-Modified` для экономии трафика.
- **Поддержка OPML:** Легкий импорт и экспорт ваших списков лент.
- **Погружение в чтение:** Чистый режим чтения, индикатор прогресса и динамическое оглавление.
- **Медиа в приоритете:** Встроенная поддержка видео YouTube и Bilibili "из коробки".

### Интеграция с ИИ
- **Умные сводки:** Автоматическое генерирование сводок статей с использованием API OpenAI или Anthropic.
- **Перевод в один клик:** Нативный перевод целых статей без выхода из приложения.
- **Правила автоматизации:** Создавайте мощные правила на базе ИИ (например, "Автоматически помечать статьи с оценкой релевантности ИИ > 80 звездочкой").

### Нативный десктопный опыт
- **Локальное кэширование:** Собственный протокол `rss-media://` надежно кэширует изображения и потоковое видео для оффлайн-работы.
- **Высокая производительность:** Виртуализированные списки (`react-virtuoso`) легко справляются с тысячами статей.
- **Для продвинутых пользователей:** Полноценные глобальные сочетания клавиш для навигации без мыши.
- **Темы:** Поддержка светлой и темной тем, соответствующих системным настройкам.
- **Многоязычность:** Доступно на арабском, китайском, английском, французском, русском и испанском языках.

## Технологии

Этот проект использует современные веб- и системные языки для предоставления легкого нативного приложения:

- **Ядро:** [Tauri 2.0](https://v2.tauri.app/) + [Rust](https://www.rust-lang.org/)
- **Фронтенд:** [React 18](https://react.dev/) + [TypeScript](https://www.typescriptlang.org/)
- **Стилизация:** [Tailwind CSS](https://tailwindcss.com/)
- **Управление состоянием:** [Zustand](https://docs.pmnd.rs/zustand/)
- **База данных:** SQLite (через встроенный `rusqlite`)

## Разработка

### Предварительные требования
- [Node.js](https://nodejs.org/) (v18+)
- [Rust](https://www.rust-lang.org/tools/install) (v1.70+)

### Быстрый старт

```bash
# 1. Клонировать репозиторий
git clone https://github.com/your-username/rss-reader.git
cd rss-reader

# 2. Установить зависимости фронтенда
npm install

# 3. Запустить сервер разработки (Фронтенд + Rust бэкенд)
npm run tauri:dev
```

### Полезные команды

| Команда | Описание |
|---------|-------------|
| `npm run tauri:build` | Сборка релизного бандла приложения |
| `npm run dev`         | Запуск только фронтенда Vite (без Rust бэкенда) |
| `npm test`            | Запуск тестов фронтенда (Vitest) |
| `cargo test`          | Запуск тестов бэкенда (Rust) |
| `npm run lint`        | Запуск ESLint |

### Обзор архитектуры
В приложении используется четкое разделение обязанностей с помощью IPC Tauri (`invoke`):
- `src-tauri/src/db/`: Модульные операции SQLite (ленты, теги, группы, правила).
- `src-tauri/src/feed/`: Парсинг лент с использованием `feed-rs`.
- `src/stores/`: Хранилища Zustand с сохранением в LocalStorage для глобального состояния.
- `src/components/`: Модульные компоненты React для пользовательского интерфейса.
